# On Context Evolution

To push the capabilities of LLMs, a multi stage process must execute as a reflext to user input. Reflexes then spawn mechanical and agentic processes alike, with the goal of evolving what is mechanical into more relevant context, optimizing the speed of context engineering, and accuracy. Budgets are assigned to each proceess.

## Evaluation

Context evolution measurements can be made from temporal context sets, like past conversations, project files, cortex, and all the tools your AI uses. We should start with including 3-5 real world coding tasks, and where inefficiences lie.

E.g.
let s represent the subject of multi session AI tool usage, with context becoming available temporally.

Looking to build `s`
*tool builds a prototype*
*cortex learns, thinks, and dreams about `s`
Lets reach production readiness for `s`
*tool works to make the project production ready*
*cortex suggests context through fast mechanical lookup, pre-processed agentically*
*cortex evolves this mechanical context over time*
Solve a critical bug
*tool and cortex work together to fix defect*
Retro the project
*tool suggests internal improvements and cortex provides past relevant context*
Make a scalability improvement
*tool makes improvements while cortex context evolves with it*
Major change
*tool makes changes with cortex increasing the usage of new patterns for more effective migration*
Observability milestone reached
*tool is working within much larger project context, and must adapt with cortex*

If Context Evolution could be applied to the software development lifecycle, could real word productivity be increased? The context evolution eval ought to consider the entire SDLC as its test bed, and assert that agentic reflection of context on demand can be used to guide higher quality AI augmented software development.

The evolution of context ought to be tuned through some parameters:
1. Hint size. How much context should be provided by context evolution to avoid context pollution.
2. Decay. Baseline value for how quickly Think and Dream ought to decay in budget over time. 
3. Depth. Budgets for Reflect, Think and Dream
